Dokumentation av det här APIet

Routes

  • POST /users - skapar en ny användare, accepterar JSON objekt på formatet: {"firstname": "", "lastname": "", "userId": "unikt namn", "passwd": ""}. userId är obligatoriskt.
  • PUT /users/:id - updatera användarattributer, accepterar JSON objekt på formatet: {"firstname": "", "lastname": "", "userId": ""}. alltingen firstname eller lastname är obligatoriskt.
  • POST /login - logga in, accepterar JSON objekt på formatet; returnerar en token som krävs för visa routes: {"firstname": "", "lastname": "",}. userId och passwd är obligatoriskt.
  • GET /getusers - visar regristerade användare, accepterar JSON objekt på formatet: {"token": ""}. token är obligatoriskt.
  • GET /getuserme - visar den inloggade användares attributer, accepterar JSON objekt på formatet: {"userId": "", token": ""}. token är obligatoriskt.
